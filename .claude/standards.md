# Blik Development Standards

## Code Style

### CSS
- NO inline styles in HTML/templates
- All CSS in structured CSS files
- Use shadcn/ui conventions and utilities
- Follow shadcn design patterns

### JavaScript
- Use vanilla/plain JavaScript where possible
- NO `alert()` or `prompt()` - use modals and proper UI components
- Progressive enhancement approach

### Content
- NO emojis anywhere in code, UI, or commit messages
- Zero noise, 100% signal
- Professional, technical communication only

### Django
- Follow Django best practices
- Keep views clean and focused
- Use class-based views where appropriate
- Proper model organization

### Documentation
- Only essential documentation (README, REQUIREMENTS)
- No duplicate documentation
- Focus on signal, not noise
